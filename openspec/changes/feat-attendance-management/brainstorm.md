## 2. 問題定義
- **核心目標**: 建立一個出缺勤管理系統，支援員工打卡、請假、加班申請與簽核流程
- **利害關係人需求**:
  - 管理者：管理使用者帳號、檢視出勤報表
  - 主管：簽核下屬假單與加班申請
  - 員工：每日打卡、申請請假/加班、查看假別額度
- **技術限制與邊界**:
  - 前端：Vite + Vue.js 3
  - 後端：Express + TypeScript
  - 資料庫：本機 MySQL

## 3. 解決方案方案探索
### 方案 A: 前後端分離 + REST API（Monorepo 目錄組織）
- **優點**: 職責分離清楚、共用 TypeScript 型別、前後端獨立開發
- **缺點**: 需處理 CORS、JWT session 管理
### 方案 B: 全端 SSR (Nuxt.js)
- **優點**: SEO 友善、前後端一體
- **缺點**: 與指定 Express 後端衝突、複雜度高
### 方案 C: 獨立前後端倉庫
- **優點**: 完全解耦
- **缺點**: 型別無法共用、版控分散

## 4. 最終決策路徑
- **選定方案**: 方案 A — 前後端分離 + REST API，Monorepo 目錄組織
- **關鍵理由**: 符合指定技術棧、共用型別提升一致性、單一 repo 方便管理
